# Roadmap

This roadmap is intentionally practical.

It separates:

- what should happen now for the plant pilot
- what should happen over the next 45 days for a stronger final delivery
- what belongs to later productization

Supporting docs:

- Honest engineering gap review: [docs/engineering-gap-review.md](docs/engineering-gap-review.md)
- Final 45-day plan: [docs/final-45-day-plan.md](docs/final-45-day-plan.md)

## Immediate priority: plant pilot readiness

- deploy to the plant/server PC successfully
- verify continuous polling with real meters
- verify frontend dashboard against live data
- verify restart/recovery procedure
- verify backup and recovery basics

## Next 45 days: professional internship-final build

- improve dashboard and meter-management UX
- improve report/export quality and clarity
- add lightweight operator help/instructions content
- tighten Windows packaging/deployment workflow
- improve handover and maintenance quality

## Mid-term productization

- installer-based Windows software packaging
- better operational tooling and diagnostics
- stronger authentication/authorization
- backup and retention automation
- upgrade and support workflow

## Longer-term product growth

- per-site or multi-location support
- richer alert escalation channels
- deeper historical analytics
- device management and remote health visibility

## Explicit non-priorities right now

- cloud migration
- Kubernetes or heavy container orchestration
- major database redesign
- premature microservice refactor
- enterprise identity platform
